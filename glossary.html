Definitions

Below is a quick glossary of specific terms and terminology referenced in this site.

Term Descrption Additional Notes
API Application Programming Interface (API) is a set of definitions and protocols for integrating software. APIs in an enterprise can be categorized as edge, utility, and domain APIs. Edge APIs act as the gateway to the enterprise systems while domain APIs expose the internal systems to the integration layer. Utility APIs contain the integration logic acting as the glue between edge APIs and the internal systems. Web article: What are the different types of APIs?
API-First Approach In an API-first approach, we start with the API design and then move into the implementation of APIs as well as the integration logic. Since we begin with API design, we call this the API design-first or, for brevity, API-first approach. This is also known in the Enterprise Integration Patterns canon as the 'canonical data model' message transformation pattern . Web article: A Seven-Step Guide to API-First Integration
DEMS Digital Evidence Management System.
HTTPS Hypertext transfer protocol secure (HTTPS) is the secure version of HTTP Traffic is encrypted using TLS.
Integration-First Approach An integration-first approach is a traditional bottom-up approach where we build the integration logic first on top of backend services and then expose the integrated services (also known as composite services) as managed APIs to users. Web article: A Seven-Step Guide to API-First Integration
Integration Pattern An integration pattern refers to any standardized method for messaging between two IT systems. This could be relating to messaging endpoints (e.g. polling consumer), message construction (e.g. reqeuest-reply), messaging channels (e.g. publish-and-subscribe channel), message routing (e.g. content-based router), message transformation (e.g. canonical data model), and systems management (e.g. message history). Web article (from Enterprise Integration Patterns): Solving Integration Problems using Patterns
Integration Service An integration service is a secure communication component that transports and can transform data from one organization (business area or partner) to another. More info: What is an integration service?
Integration Stack A specific implementation of an integration service with key technologies and products identified.
JSON JavaScript Object Notation (JSON) is a lightweight data-interchange format that is easy for humans to read and write.
JUSTIN BCPS Case Management System.
JWT Json Web Token (JWT) is an open, industry standard RFC 7519 method for securely transmitting information between parties as a JSON object, and which is digitally signed. Web article: JSON Web Tokens vs Oauth 2.0
OAuth / OAuth 2.0 OAuth and OAuth 2.0 are the industry-standard protocols for authorization. OAuth allows you to use a 3rd party application to access a specific access-controlled digital content or service (e.g. photos stored on Google Drive) without you needing to share your Google email and password with the 3rd party application. OAuth 2.0 focuses on client developer simplicity while providing specific authorization flows for web applications, desktop applications, mobile phones, and living room devices. Web article: OAuth 2.0
REST API Representational State Transfer (REST) is a set of architectural constraints used to design stateless APIs. Web article: Event-driven vs RET API interactions
Streaming API Streaming API (sometimes referred to as event-driven, or realtime API) interaction patterns differ from REST API. Instead of Web articles:
  • Event-driven vs RET API interactions
  • The Realtime API Family
TLS Transport Layer Security (TLS) is a security protocol designed to facilitate privacy and data security for communications over the Internet.
