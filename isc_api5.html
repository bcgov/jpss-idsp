Check Case User Access Approval (REST API)

This is a callable domain REST API from the Court Case User Access Lookup Service. This operation allows the calling application to determine if a specific BCPS user can access the indicated court caseand its associated case materials.


Below is a solution concept diagram depicting the Court Case Management Services and the initial solution integrating three enterprise applications: JUSTIN, CORNET and DEMS.


The User Access Lookup Service allows calling applications to check whether a given BCPS user has access to a particular case.

Technical Specification

API Features

Feature Details
OpenShift Namespace AG PSSG JADE Platform (cef5dd)
API Type REST
REST API Service ccm-user-access-lookup-service
REST HTTP Method GET
REST HTTP URI /bcpsUsers/{userId}/courtCases/{casenumber}/checkCaseUserAccessApproval
Service Provider Application JUSTIN
Service Consumer Application DEMS

 


API Schema

This schema is defined using the Open API 3.0.1 specification. It can be viewed using an online OpenAPI editor: e.g., https://editor.swagger.io

---
openapi: 3.0.2
info:
  title: Check Case User Access Approval API
  version: 0.0.1
  description: This operation allows the calling application to check the the access permission of the specific court case for the identified user.
paths:
  /bcpsUsers/{userId}/courtCases/{caseNumber}/checkCaseUserAccessApproval:
    get:
      parameters:
      - name: userId
        in: path
        description: user id
        required: true
        schema:
          type: string
      - name: caseNumber
        in: path
        description: case number
        required: true
        schema:
          type: string
      responses:
        200:
          content:
            application/json:
              schema:
                type: object
                properties:
                  approved:
                    type: string
                    example: 'true'
          description: Case User Access Approval Status
        401:
          description: Not authorized
          content: {}
        404:
          description: User id or case number not found
          content: {}
      operationId: checkCaseUserAccessApproval
